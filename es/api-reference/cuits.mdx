# 🏢 GET /cuits - Gestión de CUITs

Este endpoint te permite obtener la lista de CUITs asociados a tu cuenta y gestionar su información.

## URL Base

```
https://facture.ar/api/cuits
```

## Autenticación

Todas las peticiones requieren autenticación usando:

- **Sesión Web**: Para aplicaciones basadas en navegador
- **API Key**: Para integraciones servidor a servidor

Incluye tu API key en el header `x-api-key`:

```bash
x-api-key: tu-api-key-aqui
```

## Parámetros de Consulta

| Parámetro | Tipo | Requerido | Descripción | Ejemplo |
| --------- | ---- | --------- | ----------- | ------- |
| `status` | string | ❌ | Filtrar por estado del CUIT | `"active"` |
| `type` | string | ❌ | Filtrar por tipo de CUIT | `"monotributo"` |
| `includeDefaults` | boolean | ❌ | Incluir valores por defecto (por defecto: false) | `true` |

## Campos de Respuesta

| Campo | Tipo | Descripción |
| ----- | ---- | ----------- |
| `id` | string | ID único del CUIT |
| `cuit` | string | Número de CUIT (formato: XX-XXXXXXXX-X) |
| `businessName` | string | Razón social |
| `status` | string | Estado del CUIT (active, inactive, pending) |
| `type` | string | Tipo de CUIT (monotributo, iva_responsable, exento) |
| `category` | string | Categoría fiscal (para monotributo: A, B, C, etc.) |
| `isDefault` | boolean | Si es el CUIT por defecto |
| `defaults` | object | Valores por defecto del CUIT |
| `afipData` | object | Datos específicos de AFIP |
| `createdAt` | string | Fecha de creación (ISO 8601) |
| `updatedAt` | string | Fecha de última actualización (ISO 8601) |

## Estados de CUIT

| Estado | Descripción |
| ------ | ----------- |
| `active` | CUIT activo y configurado |
| `inactive` | CUIT inactivo |
| `pending` | CUIT en proceso de configuración |

## Tipos de CUIT

| Tipo | Descripción |
| ---- | ----------- |
| `monotributo` | Monotributista |
| `iva_responsable` | IVA Responsable |
| `exento` | Exento de IVA |

## Ejemplos de Uso

### Obtener Todos los CUITs

```bash
curl -X GET https://facture.ar/api/cuits \
  -H "x-api-key: tu-api-key"
```

### Obtener CUITs Activos con Valores por Defecto

```bash
curl -X GET "https://facture.ar/api/cuits?status=active&includeDefaults=true" \
  -H "x-api-key: tu-api-key"
```

### Filtrar por Tipo de CUIT

```bash
curl -X GET "https://facture.ar/api/cuits?type=monotributo" \
  -H "x-api-key: tu-api-key"
```

### JavaScript/Node.js

```javascript
// Obtener todos los CUITs con valores por defecto
const response = await fetch('https://facture.ar/api/cuits?includeDefaults=true', {
  method: 'GET',
  headers: {
    'x-api-key': 'tu-api-key',
  },
});

const result = await response.json();
console.log('CUITs:', result.data);

// Encontrar el CUIT por defecto
const defaultCuit = result.data.find(cuit => cuit.isDefault);
console.log('CUIT por defecto:', defaultCuit);
```

### Python

```python
import requests

# Obtener CUITs activos
response = requests.get(
    'https://facture.ar/api/cuits',
    headers={
        'x-api-key': 'tu-api-key'
    },
    params={
        'status': 'active',
        'includeDefaults': True
    }
)

result = response.json()
print('CUITs activos:', result['data'])

# Encontrar el CUIT por defecto
default_cuit = next((cuit for cuit in result['data'] if cuit['isDefault']), None)
print('CUIT por defecto:', default_cuit)
```

## ❓ Solución de Problemas

<Accordion title="Error: No se encontraron CUITs">
**Causa**: No tienes CUITs configurados en tu cuenta.

**Solución**:

1. Verifica que hayas configurado al menos un CUIT
2. Asegúrate de que el CUIT esté activo
3. Revisa el estado de configuración en tu dashboard
</Accordion>

<Accordion title="Error: CUIT inactivo">
**Causa**: El CUIT está inactivo o en proceso de configuración.

**Solución**:

1. Verifica el estado del CUIT en tu dashboard
2. Completa la configuración si está pendiente
3. Contacta soporte si el CUIT está inactivo
</Accordion>

<Accordion title="Error: No autorizado">
**Causa**: API key inválida o sesión expirada.

**Solución**:

1. Verifica que tu API key sea válida
2. Asegúrate de incluir el header `x-api-key`
3. Regenera tu API key si es necesario
</Accordion>
