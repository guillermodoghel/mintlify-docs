# ⚙️ CUIT Defaults - Configurar Valores por Defecto

La API de CUIT Defaults te permite configurar valores por defecto para tus CUITs que se utilizarán al crear facturas.

## URL Base

```
https://facture.ar/api/cuits/{cuitId}
```

## Autenticación

Todas las peticiones requieren autenticación usando:

- **Sesión Web**: Para aplicaciones basadas en navegador
- **API Key**: Para integraciones servidor a servidor

Incluye tu API key en el header `x-api-key`:

```bash
x-api-key: tu-api-key-aqui
```

## POST /api/cuits/{cuitId}/default-concept

Establece el tipo de concepto por defecto para un CUIT. Este valor se usará al crear facturas si no se especifica `concepto`.

**Endpoint**: `POST /api/cuits/{cuitId}/default-concept`

### Parámetros de Ruta

| Parámetro | Tipo   | Requerido | Descripción   |
| --------- | ------ | --------- | ------------- |
| `cuitId`  | string | ✅        | UUID del CUIT |

### Cuerpo de la Petición

```json
{
  "defaultConcept": 2
}
```

### Parámetros

| Parámetro        | Tipo   | Requerido | Descripción                                          | Ejemplo |
| ---------------- | ------ | --------- | ---------------------------------------------------- | ------- |
| `defaultConcept` | number | ✅        | Tipo de concepto (1=Productos, 2=Servicios, 3=Ambos) | `2`     |

### Respuesta

#### Éxito (200)

```json
{
  "message": "Default concept updated successfully"
}
```

#### Respuestas de Error

##### Error de Validación (400)

```json
{
  "error": "Default concept must be a positive number"
}
```

##### CUIT No Encontrado (404)

```json
{
  "error": "CUIT not found"
}
```

##### Error del Servidor (500)

```json
{
  "error": "Failed to update default concept"
}
```

## Cómo Funcionan los Valores por Defecto

Al crear facturas a través del endpoint `/api/invoices`, se aplican los siguientes valores por defecto:

1. **Si no se proporciona `concepto`**: Usa el valor `default_concept` del CUIT
2. **Si no se proporciona `ptoVta`**: Usa el valor `default_pos` del CUIT
3. **Si no se proporciona `invoiceType`**: Usa el valor `default_invoice_type` del CUIT, o se determina automáticamente basado en el tipo de CUIT (Monotributista → Factura C, IVA Responsable → Factura A, otras categorías → determinado por datos de AFIP)

### Jerarquía de Valores por Defecto

```javascript
// Al crear una factura, los valores por defecto se aplican en este orden:
const finalValues = {
  concepto: input.concepto || cuit.default_concept || 1,
  ptoVta: input.ptoVta || cuit.default_pos || 1,
  invoiceType: input.invoiceType || cuit.default_invoice_type || getInvoiceTypeForCuit(cuit) || 11,
  // ... otros valores por defecto
};
```

| Código             | Descripción                                  |

## ❓ Solución de Problemas

<Accordion title="Error: CUIT no encontrado">
**Causa**: El CUIT especificado no existe o no pertenece al usuario.

**Solución**:

1. Verifica que el `cuitId` sea correcto
2. Asegúrate de que el CUIT esté configurado y activo
3. Consulta la lista de CUITs disponibles
   </Accordion>

<Accordion title="Error: Valor de concepto inválido">
**Causa**: El valor del concepto no es válido.

**Solución**:

1. Usa solo valores válidos: 1 (Productos), 2 (Servicios), 3 (Ambos)
2. Verifica que el valor sea un número entero positivo
3. Consulta la tabla de tipos de concepto
   </Accordion>

<Accordion title="Error: Punto de venta inválido">
**Causa**: El número de punto de venta no es válido.

**Solución**:

1. Usa valores entre 1 y 9999
2. Verifica que el punto de venta esté habilitado en AFIP
3. Asegúrate de que el valor sea un número entero positivo
   </Accordion>
